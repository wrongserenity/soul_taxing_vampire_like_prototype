# PROJECT.md — Top-Down Survivor (Godot) / Web (itch.io)

Этот файл — **единый источник истины** для проекта: структура репозитория, назначение папок, правила разработки и особенности Web-сборки (itch.io).

---

## 1) О проекте

**Жанр:** Top-Down Survivor (vampire-like)  
**Движок:** Godot  
**Целевая платформа №1:** Web (itch.io)  
**Доп. платформа (опционально):** Desktop

Цели структуры:
- масштабируемость (рост контента без хаоса);
- удобная навигация (быстро найти любую часть);
- слабая связанность (системы не завязаны друг на друга напрямую);
- Web-first подход (без платформенных сюрпризов).

---

## 2) Принципы организации

### 2.1 Feature-based + Layers
Проект организован по смыслу:
- **feature-модули** (player, enemies, ui, levels),
- **системы** (systems) как “движок игры”,
- **ресурсы** отдельно от логики (assets).

### 2.2 Сцена рядом со скриптом
Если есть `X.tscn`, то рядом лежит `X.gd` (или основной скрипт сцены).

### 2.3 Web-first
Любая логика должна:
- работать без `OS.execute`;
- сохранять данные только в `user://`;
- не полагаться на автозапуск аудио;
- не зависеть от платформенных путей.

---

## 3) Структура репозитория (корень)

```text
/
├── .github/              # CI, шаблоны issue/PR (опционально)
├── assets/               # Все игровые ресурсы (БЕЗ логики)
├── autoload/             # Глобальные singleton-скрипты (Project Settings → Autoload)
├── data/                 # Игровые данные (баланс, таблицы, json/cfg)
├── documentation/        # Документация (если нужна отдельно от этого файла)
├── effects/              # VFX/Particles/Shader эффекты как модуль
├── enemies/              # Враги как feature-модули
├── levels/               # Уровни/арены/карты
├── player/               # Игрок как feature-модуль
├── scenes/               # Базовые переиспользуемые сцены (не привязанные к feature)
├── scripts/              # Общие утилиты/библиотека проекта
├── systems/              # Игровые системы (spawn, wave, damage, upgrades, save)
├── ui/                   # UI сцены/экраны/виджеты
├── web/                  # Web-специфичное (html shell, favicon и т.д.)
├── project.godot
└── README.md             # Короткое описание для GitHub/itch, без дубля структуры
```
